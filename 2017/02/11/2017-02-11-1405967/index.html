2/9 全栈日记-时间利用思考

发表于 2017-02-11 | 分类于 编程 |

Objective

关于今天的课程，你记得什么？
完成了什么？

工作加班，回来的太晚就在复盘教程

思考了下面几点：

  • 时间记录表格的应用，需要花多久记录？你能坚持多久呢？
  • 关于课程复盘，为什么不直接把教程重写一遍？看教程和写下来完全是2个事情，就和看书一样，看的时候觉得自己懂了，真的能提取出来么？不能！一样的道理嘛。为什么没有这么做呢？除了工作，真的没时间，先把教程完成一遍再说，这个是后面的事，而且是肯定要做的事
  • 编程知识的记录，有点时候新的想法，新的写法，有记录，但是太散了，要及时复盘总结，不然会忘记！怎么能忘不了呢？高频刻意练习，现在这个时间段不允许。那么在记录后要加进todolist中给自己个提醒，这是我新发现的，要抽个时间去刻意练习达到真正学会，真正学会的标准就是有一天发现自己忘不了了！

Reflective

你要如何形容今天的情绪
今天的高峰是什么？
今天的低点是什么？

虽然加班了，但是在回来的时间利用睡前的时候思考了上面2件事，也很开心。没有盲目的投入课程学习中给自己增加焦虑感，因为我知道学得会时迟早的事，不过现在不是合适的时间，既然时间不合适，那这点时间能干什么呢？想一下今天的orid、概念、方法论？

interpretive

我们今天学到了什么？
今天最重要的领悟是什么？

时间有限的情况下，能做什么事呢？预估时间，达成目标。思考出实质性的东西也是很有用的。

在正确的时间做正确的事情。

Decisional

我们会如何用一句话形容今天的工作？
有哪些工作需要明天继续努力？

虽然没有课程进度，不过也有今天的进步

  • 时间开销记录
  • 课程进度抓紧，明天晚上没事的话就完成这周的课程，剩余orderlist不多，争取周末再练一遍。不能练3遍就起码练2遍吧！
