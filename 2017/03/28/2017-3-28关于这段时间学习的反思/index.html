关于这段时间学习的反思

发现自己的学习效率降下来了，去看了xdite的博客，反思总结下。

这段时间的问题：

  1. 没有做具体的项目，没有了具体的反馈，没有了smallwin。导致效率明显下降，跑去看各种的rule book,陷入了what->why死亡循环！
  2. 元学习课没有具体实践，学了最厉害的方法不去实践，只是图个脑爽而已。
  3. 面对一堆要学的东西（部署、API、testing、js），对这张混乱的状态没处理好，再加上不停的死循环->no small win->拖延！这种大的死循环！

还好无意中翻到xdite-学习这回事，真的特别感谢xdite，去年做的最明智的决定就是认识她，反思了上面的三条。

再复习下（敲一遍）：

土拨鼠候群症：简单的说，一个人会不断的重复自己犯过的错，每次都会在类似的事上一直犯同样的错误，循环自杀！

通常当事人都不会觉察到自己有这种倾向，只会觉得自己只是运气不好。但是旁观者却会觉得触目惊心：干，他又来了，妈的三个月之后我又要帮他收尸了。

当你看到朋友老是因为类似的决策去死，自己却无法阻挡时，那种痛苦真的令人难以承受。

回头想想，自己会不会有这种毛病？我这段时间就是典型，所以我为了记住，就把这种状态称为死亡循环。

其实每个人都或多或少都有这种倾向，症状比较轻点，通常都是比较了解自己的人。了解自己越透彻的人，越不会有这张问题。

不了解自己的人才会觉得自己一天衰到晚，然后自怨自艾，嫉妒他人，继续循环的徒劳无功。

看到很多年轻时代觉得就这样萎靡下去了，只能感叹。

人在几年间进步的差异，就真的卡在这里而已。

每天练习才是变强的关键

  • 我没有刻意在练，而是每天无意识的玩这些新东西，无形之中其实就是一种强度非常大的练习。这样的日子我过了四五年。
  • 回顾所以我练不起来的东西，没有一项是经过强大的练习后，还搞不起来的例子。
  • 因为我付了钱，不练就是把钱丢水里。
  • 他们的知识取得的成本都相对很低廉，既然廉价，自然就没有得到那么高的重视

如何飞速学一门技能

Priciple

  1. Focus on language content that is relevant to you
  2. Use your new language as a Tool To Communicate From Day 1
  3. Understand the Message you will un consciously Acquire the Language
  4. Physiologica Training
  5. Physicological State Matters!

Action:

  1. Listen A Lot
  2. Focus on getting the meaning first( input -> output)
  3. Start Mixing
  4. Focus on the Core
  5. Get a Language Parent
  6. Copy the face
  7. Direct Connect to Mental Images

状态：ORID就是在记录这个状态，发现自己的状态和学习效果！

心理生理的状态，如果你伤心、生气、担心、沮丧、你将不能学会！如果你是在一个开心、放松、好奇的大脑状态下，你将很快学会！你需要tolorate ambiguity! tolorate perfact!

xdite平时自己用的方法

  1. 找到目标教程里面和现在要解决的眼前状况有关系的部分
  2. 尽情的读相关的教材—泡脑子
  3. 中间有看不懂的也没关系，不需要停下了弄懂它。只享受弄懂的部分
  4. 过滤出Pattern，掌握核心几个技巧
  5. 利用这些核心技巧开始混合练习
  6. 招数使出去，发现歪掉了就回去看别人怎么做的，纠正自己
  7. 用映射技巧把新技能跟自己已经了解的知识做Mapping变成自己原有的一部分

就是看完太极剑演示再忘记太极剑招式只记得剑义的意思。

学习一个新领域的知识的最佳方法和时间各是什么？

99%的人学不会一项技能的原因：挫折感太大。这不是我的原因，我不会管这些挫折感，因为知道这些挫折感是正常的，学新东西出错再正常不过了。我的原因1是触发了 Why -> What -> Why -> What 死亡循环。

正常的循环应该是： What （重复 3 遍） -> Why -> How （独立重制步骤）。

重复3遍的定义是：

  1. 抄，搞懂input->output
  2. 记，自己打，记录错误
  3. 背，自己默写，写不出来就重复2

原因2是没有重复3遍！
