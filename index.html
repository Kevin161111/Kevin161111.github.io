git clone

git clone下来的专案默认master分支，不会有其他的分支下来

git clone https://github.com/twbs/bootstrap.git 下来的文件夹默认就是bootstrap

git clone https://github.com/twbs/bootstrap.git my_bootstrap下来的文件夹名字为my_bootstrap

git clone https://github.com/twbs/bootstrap.git -b v3-dev只是将v3-dev这个分支clone 下来

git branch

git branch -a会列出本地和远程的所有分支

git branch -r只会列出远程的

git branch列出本地的

git checkout

git checkout origin/step4会切换到step4的最后一次提交，并以hash值为分支名称

git checkout origin/step4 -b newbranch本地建立newbranch 分支并将远程step4复制到里面

git pull

git pull origin step4 将step4的分支pull 到本地的当前分支

git push

git push origin :step4删除远程的step4分支

Ajax是什么？如何实现？

Ajax（Asynchronous JavaScript and XML）异步的JavaScript 与XML技术，不需要整页替换，只需要更新部分网页。大大加速UI反应速度。常见回传Json 和Script 格式

用Json格式，必须写自定义的JavaScript去处理Ajax的发出和接受处理。JavaScript 强者或队伍中有前端工程师会这么写。

另一种是用Script格式，服务器回传JavaScript，浏览器拿到后直接执行。这种方式Rails有内建Jquery-ujs gem，会帮忙绑好事件去处理Ajax发送和接收。这是比较简单的做法，对jQuery技能的要求也比较低。

git reset-reset current HEAD to the specified state

HEAD 是什么？

第一行那个HEAD就是对应目前的commit位置，它是HEAD~0，它下面那个commit是HEAD~1,依次下排

如果commit错了，想撤回

git reset -- soft HEAD~1结果如下：

如果要撤销2个就写HEAD~2，结果如下：

可以看到只是HEAD指向的位置不同了，代码并没有改变

说明这只是撤销commit而已。

如果要改变呢？看下面

  • –mixed：默认参数，git reset --mixed HEAD~1等同于git reset HEAD~1。实现的功能是撤销commit的同时，撤销git add .的内容

  • –soft: 撤销commit ,不撤销 git add.
  • —hard: 删除改动的代码，撤销commit ，撤销 git add. 回到上次commit 的状态

除了用在commit上，还可以这么用：比如误删了文件，也是可以通过git reset -- hard HEAD~2找回来的。里面的HEAD~2指的是你要返回的位置，要调整的，比如3、4、5。。。。当然这个HEAD~n，也是可以用前面的hash代替的，比如git reset -- hard 6cba643

修改commit的注释：

  • git commit --amend会进入vim编辑器，直接改最上面的就好了。ESC退出编辑模式进入命令模式，输入:wq存盘退出

详细的介绍在这里How to write a git commit message

  1. Seperate subject from body with a blank line
  2. Limit the subject line to 50 characters
  3. Capitalize the subject line
  4. Do not end the subject line with a period
  5. Use the imperative mood in the subject line
  6. wrap the body at 72 characters
  7. Use the body to explain what and why vs.how
