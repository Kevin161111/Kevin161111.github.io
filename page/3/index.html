2/20-全栈日记-加班，写点别的

objection

关于今天的课程，你记得什么？
完成了什么？

今天除了听Xdite老师的课程之外，加班在做别的事。听了老师的课，对课程有了更大的信心。对之前两个月的学习做了一个复盘，也明白了老师设置课程的初心。

Reflective

你要如何形容今天的情绪
今天的高峰是什么？
今天的低点是什么？

平静，听到老师关于教学的后续很有信心。

Interpretive

我们今天学到了什么？
今天最重要的领悟是什么？

最大的领悟其实来自想通了一件事。成为专家，只要坚持足够的时间积累就可以了。这句话有毒。大部分都知道，而且很多都做了，结果为什么后面没成为专家呢？因为长久以来坚持做的东西都是自己熟悉的呀！并没有突破舒适区呀？刻意练习的本质除了刻意，另外就是练习的内容要进入学习区，也即是老师讲的觉得有点难，但是可以学，没有进入太多的挫折感。
这让我想到课程学习上不要一直看别人的教程，想办法变成自己的，写出成为自己的东西。一旦有自己发现自己想要看别人的教程，就是在走简单抄教程的路上，没有自己的思索，做伸手党的后果就是伸手党陷阱：别人给的是解决了自己的问题，但同时也主动丧失了自我思索，自我检索的能力！

Desicisional

我们会如何用一句话形容今天的工作？
有哪些工作需要明天继续努力？

明天要抓时间做魔改大赛。Xdite说时间不够是因为不是刚需！太精辟了，写下来贴到了电脑上。践行才是王道。

什么是userstory？

发表于 2017-02-20 | 分类于 编程 |

userstory是工程师一个很重要的能力，站在用户的角度，用简洁的语言描述一个功能。
为什么这个能力很重要？从实际的需求转换成实际的功能是通过它实现的，功能到实际的代码也是用它来做出来的
这是工程师思维中很重要的一个存在。也是后续用它来进行项目管理的一个目标。

严谨、漂亮、详细都不是他的要求，严谨的文件本身是一个目标而不是开放的功能，语言本身就是模糊的，用详细的文件去描绘会造成“确定性的幻想”以为规划好了就无需沟通了，导致最终做出不符合预期的功能。

实例：

  1. 身为消费者，我要很方便的在这个网络商店买到东西
  2. 在前台买到东西并结账
  3. 加入购物车，生产订单
  4. 生成订单后，填写地址并付款
  5. 结账后，生成订单确认信件
  6. 店家收到已经付款的订单通知
  7. 货架上的库存必须随之减少

2/19本周两最

发表于 2017-02-19 | 分类于 orid |

本周学到最有用的东西：

这周学到最有用的东西是提取练习，之前只是名词，现在我把它变成自己的行动了，总结套路如下：

  • 前提是自己对教程有了感觉，马上写提取练习，这样会加深自己的理解，形成自己的东西
  • 对不懂得东西，有时间就多练几遍形成肌肉记忆，以后就理解了，到时再写成提取练习
  • 对自己学到的东西一定要记下来，因为记忆是有时间的，说不定下次就忘了，这个时候就不用再去翻google交时间税了，翻自己的教程就可以了。

本周最大的坑：

在不正确的时间做不正确的事就是一连串的不幸运。在踏入heroku和aws这个坑的时候，由于想快点把功能实现以进行魔改大赛，结果在晚上12点还在解bug，忘记了自己debug半个小时就应该放弃的原则。导致挫折感很强烈，虽然最终解决了，但是反思下来如果自己不是这么晚deploy会不会就明天解决说不定思路更清晰呢？

复习下幸运公式;幸运是乘法，幸运就是好的结果+意想不到的放大与连锁

如何常年保持幸运？

  1. 反复的做，只有反复的做，才知道哪些是控制变量
  2. 发现低分立马改正
  3. 把你的资源全部灌注在你的幸运model上
  4. 多学些放大器技能，比如编程和英语

深圳线下meetup总结

发表于 2017-02-19 | 分类于 编程 |

深圳线下MeetUp总结：

作为组织者，总结下：

  • 下次要找个好的地点，提前人数统计完，位置要足够多，办公室的位置有点挤了
  • 关于地点已经和小伙伴们讨论了每个人回去自己找一个觉得适合的，然后一起决定。
  • 让大部分的同学都已经有了自己的小伙伴，不同进步的人形成小组讨论
  • 鼓励更多的人发言，让进度快的同学多做学习套路的分享。
  • 沟通下来最大的收获是学的好的前提都是有一个规律的作息，利用项目管理的方式对时间进行管理，这也是我最近体会比较深的一点
  • logdown要坚持写，写的多了就停不下来了。
  • 对教程的提取联系，要在对教程有感觉的情况下进行，如果是一点都看不懂的，形成肌肉记忆就好。
  • 坑踩得越多越好，把踩得坑的多了就成自己的东西了
  • 学习这东西，开始的时候可以用别人的教程，别人的套路，最终是要提取练习成自己的套路的。
  • 课程中功能的达成可能由于版本的原因形成不同的写法，这个是因为版本的原因，至于采用哪种写法，看习惯。也可以在学习中根据自己的习惯对相同的功能进行归纳总结。
  • 关于魔改大赛，参赛只是手段，目的是通过参赛熟悉前端的操作，也为日后找工作有拿得出手的东西。
  • 前端的操作以满足自己需求为主，在github-project上面按照老师分享的方法各自写出要实现的目的，实现了就拉到已完成里面去，这样就会更有目的，这也是项目管理的套路之一。
  • 魔改大赛要当成真正自己的项目来做，其实以后工作也是，任何时候的目的都应该是自己的项目，而不是为了目的而目的，当成真正能为这个社会协作做出贡献的一个产品出来。
  • 和YY老师的连线也比较顺利，下次要让小伙伴们每个都准备些问题去问，这么好的机会不要错过。
  • 关于教材的提取，最终达到自己写出userstory然后自己写出功能的地步就算真正入门了，也是教程的精华所在
  • 以后每周的meetup形成固定的时间，我们要坚持分享下去，当做一个共同进步的集体来做。
